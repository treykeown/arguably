API reference

arguably is a tool for handling command-line input in Python. It is a layer on top of argparse that lets you use a function definition and a docstring to handle inputs any way a script would require.

Any time a user setting up arguably messes up, an ArguablyException will be raised. Any time a user on the command line messes up, an error message will be printed (as is normally done with argparse).

arguably.command 

arguably.command(func=None, /, *, alias=None, help=True)

Mark a decorated function as a command. If multiple functions are decorated with this, they will be available as subcommands. If only one function is decorated, it is automatically selected and the subcommand logic will not be used.

Parameters:

Name Type Description Default
func Optional[Callable]

The target function.

 None
alias Optional[str]

An alias for this function. For example, @arguably.command(alias="h") would alias h to the function that follows.

 None
help bool

If False, the help flag -h/--help will not automatically be added to this function.

 True

arguably.run 

arguably.run(
    name=None,
    call_ancestors=False,
    always_subcommand=False,
    auto_alias_cmds=False,
    auto_alias_params=False,
    version_flag=False,
    show_defaults=True,
    show_types=False,
    max_description_offset=60,
    max_width=120,
    command_metavar="command",
    output=None,
)

Set up the argument parser, parse argv, and run the appropriate command(s)

arguably.error 

arguably.error(message)

Prints an error message and exits. Should only be used for when a value from the command line is not of the correct form.

arguably.is_target 

arguably.is_target()

Aliased by arguably.is_target. Only useful when invoke_ancestors=True, it lets a command know whether it's the main targeted command or just an ancestor of the targeted command.

arguably.subtype 

arguably.subtype(cls=None, /, *, alias)

Mark a decorated class as a subtype that should be buildable for a parameter using arg.builder(). The alias parameter is required.

arguably.arg 

arguably.arg()

Static methods for adding a modifier to a parameter. Should be used in Annotated[].

arguably.arg.required 

arguably.arg.required()

Marks a field as required. For *args or a list, requires at least one item.

arguably.arg.count 

arguably.arg.count()

Will count the number of times a flag is given

arguably.arg.choices 

arguably.arg.choices()

Allows specifying a fixed number of choices

arguably.arg.missing 

arguably.arg.missing()

Will use a value different than the default if an option is specified, but no value is given for it

arguably.arg.handler 

arguably.arg.handler()

Will use a user-provided handler to take the input string value and return an output

arguably.arg.builder 

arguably.arg.builder()

Will use the arguably builder logic instead of trying to instantiate the type from the input string

arguably.ArguablyException 

arguably.ArguablyException()

Raised when a decorated function is incorrectly set up in some way

arguably.ArguablyWarning 

arguably.ArguablyWarning()

Raised when a decorated function is incorrectly set up in some way, but arguably can continue
